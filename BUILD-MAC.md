# 🎤 노래방 — 맥용 앱(.dmg) 만들기

애플 실리콘(M1~M4) 맥북에서 더블클릭으로 실행되는 앱을 만드는 방법입니다.
**반드시 맥에서** 진행하세요. (맥 앱은 맥에서만 빌드됩니다.)

## 1. 맥에 준비물
- **Node.js** 설치: https://nodejs.org (LTS 버전)
  - 또는 터미널에서 `brew install node`

## 2. 프로젝트 옮기기
이 `noraebang` 폴더를 통째로 맥으로 복사합니다.
(`node_modules`, `dist` 폴더는 빼고 복사해도 됩니다 — 어차피 새로 받습니다.)

## 3. 빌드
맥 터미널에서 폴더로 이동 후:

```bash
cd 노래방폴더경로/noraebang
npm install          # 의존성 설치 (electron, electron-builder)
npm run dist:mac     # 애플 실리콘(arm64) 앱 빌드
```

빌드가 끝나면 `dist/` 폴더 안에 생깁니다:
- `노래방-0.1.0-arm64.dmg`  ← 이 파일을 상대방에게 전달
- `mac-arm64/노래방.app`      ← 앱 본체

> 인텔/실리콘 둘 다 돌아가게 하려면: `npm run dist:mac-universal`

## 4. 받는 사람이 설치/실행
1. `.dmg`를 더블클릭 → **노래방.app**을 응용 프로그램 폴더로 드래그
2. 처음 실행 시 macOS가 "확인되지 않은 개발자" 경고를 띄움 (서명을 안 했기 때문)
   - **앱 아이콘을 우클릭 → "열기" → "열기"** 누르면 실행됩니다.
3. 그래도 "손상되어 열 수 없음"이 뜨면, 터미널에서:
   ```bash
   xattr -cr /Applications/노래방.app
   ```
   (격리 속성 제거 — 한 번만 하면 됩니다)
4. 첫 실행 때 **마이크 권한 허용** 팝업이 뜨면 "허용".

## 참고
- 서명/공증(notarization)을 하려면 유료 애플 개발자 계정($99/년)이 필요합니다.
  지금 설정은 **무료(미서명)** 배포라 위 우클릭→열기 과정이 필요합니다.
- 마이크 모니터링 지연을 줄이던 리눅스 전용 설정(PIPEWIRE_LATENCY)은
  맥에선 자동 무시됩니다. 맥에서 개발 실행은 `npm run start:mac`.
